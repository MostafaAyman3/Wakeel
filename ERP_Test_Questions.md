# ERP Agentic AI — بنك أسئلة الاختبار (Test Question Bank)

> **الغرض:** اختبار شامل لـ M1 (Intelligence Agent) و M3 (Customer Support).
> **مبني على الداتا الفعلية** المولّدة في الـ 3 ملفات SQL.
> **تاريخ المرجع:** 21 يونيو 2026.
> **ملاحظة:** الأرقام المتوقعة تقريبية ±، لأن بعضها يعتمد على حالة الدفع وقت الاستعلام. الأهم هو **النمط** و**شكل المخرج** و**نوع الـ intent**.

---

## ⚠️ ملاحظة مهمة قبل الاختبار

الداتا فيها طبقتين زمنيتين:
- **2024** (Q1+Q2): الداتا التاريخية الأساسية — كلها متأخرة دلوقتي (90+ يوم)
- **2026** (Jan-Jun): الداتا الحديثة — فيها التنوع في الـ aging buckets

عشان كده:
- أسئلة **الأداء والمقارنة** → استخدم سنة 2024 (فيها Q1 vs Q2 كاملين)
- أسئلة **العملاء المتأخرين / Aging** → النظام هيشوف التوزيع الكامل عبر كل الـ buckets

```

**الإجابة المتوقعة:** إجمالي فواتير المبيعات في Q2-2024 ≈ **4,458,000 جنيه** (150 فاتورة). النظام يعرض رقم كبير + يقدر يقارن بالربع السابق.

identifier: ORD-2024-1567

**Confidence:** 🟢 High

### سؤال M3.2 — Invoice Dispute (عربي) ⭐ Scenario 2
```

المشكلة: الفاتورة دي غلط، أنا ما طلبتش المنتج ده
```
**Issue Type المتوقع:** `billing_dispute`
**Confidence:** 🟢/🟡 (حسب اكتمال البيانات)
**الإجابة المتوقعة:** يعرض تفاصيل الفاتورة الفعلية → يولّد رد → **Human Review إلزامي** (مش auto-send). "فريق المبيعات هيتواصل معك خلال 48 ساعة."

---

### سؤال M3.3 — Missing Data (عربي) ⭐ Scenario 3
```

المشكلة: مشكلة في توصيلة رقم DEL-999
```
**Issue Type المتوقع:** غير محدد (لا بيانات)
**Confidence:** 🔴 Low
**الإجابة المتوقعة:** Graceful Degradation: "لم نتمكن من العثور على الرقم DEL-999. يُرجى التأكد من الرقم أو التواصل مع الدعم." → Escalation + logging.

---

### سؤال M3.4 — Repeat Issue (عربي) ⭐ Scenario 4
```

المشكلة: أنا عميل قديم وعندي مشكلة متكررة في التوصيل

**Confidence:** 🟢 High

# 🎯 نصائح للاختبار

1. **ابدأ بالأسئلة المنفردة** قبل السلاسل — أسهل في عزل المشاكل.
2. **راقب الـ Intent Classification أولاً** — لو غلط، باقي الـ pipeline هيغلط.
3. **تحقق من Output Format** — هل اختار الشكل الصح؟ (رقم واحد → Metric Card مش Table).
4. **في السلاسل، راقب الـ context** — هل افتكر إننا بنتكلم عن نفس الفترة/الفئة؟
5. **الأرقام التقريبية كافية** — الأهم النمط وشكل المخرج، مش الرقم بالمليم.
6. **سجّل أي سؤال يفشل** للـ debugging — خصوصاً الـ clarification والـ edge cases.
